# Code Of Conduct

ChangeForge Skill Mesh should be a professional, practical, and welcoming project space. This code of conduct applies to project repositories, issues, pull requests, discussions, reviews, and other project-controlled collaboration spaces.

## Expected Behavior

Participants are expected to:

- Be respectful, clear, and constructive.
- Critique ideas and code without personal attacks.
- Assume good intent while still asking for evidence and validation.
- Respect project boundaries, including privacy, security, and licensing constraints.
- Keep discussions focused on improving the project.
- Give credit for work and avoid misrepresenting authorship or authority.

## Unacceptable Behavior

The following behavior is not acceptable:

- Harassment, intimidation, threats, stalking, or sustained disruption.
- Insults, slurs, demeaning language, or targeted exclusion.
- Sexualized language or imagery in project spaces.
- Publishing private information without explicit permission.
- Sharing secrets, credentials, customer data, or private repository content.
- Pressuring maintainers or contributors to bypass validation, security, licensing, or release requirements.

## Enforcement

Maintainers may remove content, close issues, reject pull requests, restrict participation, or ban participants when behavior violates this code of conduct.

Enforcement should be proportional and documented privately by maintainers when possible. Maintainers with a conflict of interest should recuse themselves from enforcement decisions for that incident.

## Reporting

Report conduct concerns to the maintainers through a private project channel. If the project is hosted on GitHub and private reporting is enabled, use the repository's private vulnerability or maintainer contact channel when the report includes sensitive information.

Do not include secrets, credentials, private keys, customer data, or private repository content in public reports.
