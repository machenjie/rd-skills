# Architecture Checklist

- Identify existing architectural pattern for the affected area.
- Check module, layer, service, and data ownership boundaries.
- Confirm dependency direction remains coherent.
- Compare simple local implementation against abstraction or service split.
- Identify scalability, latency, availability, and deployment impact.
- Check operational ownership and observability.
- Record tradeoffs and rejected alternatives.
- Define migration path and reversibility.
- Identify ADR or design documentation need.
