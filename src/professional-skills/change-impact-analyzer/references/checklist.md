# Impact Checklist

- Product behavior and user-facing flows.
- UX states: empty, loading, error, success, disabled, validation.
- Domain entities, invariants, permissions, events, state machines.
- API contracts, DTOs, error model, pagination, versioning.
- Data schemas, migrations, indexes, retention, consistency.
- Frontend, backend, integration, security, privacy, and AI surfaces.
- Tests, fixtures, mocks, contract tests, E2E, rollback tests.
- Deployment, feature flags, observability, alerts, runbooks.
- Compatibility across clients, workers, jobs, and consumers.
- Documentation and release communication.
