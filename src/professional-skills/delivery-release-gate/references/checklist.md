# Delivery Checklist

- Confirm CI status and artifact provenance.
- Verify Docker image, runtime, and dependency expectations.
- Review Kubernetes, gateway, and environment configuration.
- Verify secrets references without exposing secret values.
- Sequence migrations, backfills, flags, and code deploys.
- Define staging validation and production smoke checks.
- Choose canary, blue-green, rolling, or direct rollout.
- Define rollback triggers, steps, and owner.
- Prepare release notes and post-release monitoring window.
