# Reliability And Observability Checklist

- Identify user-visible and system SLIs.
- Define latency, throughput, error, and saturation expectations.
- Review concurrency, locking, queue depth, and rate limits.
- Define timeout, retry, circuit breaker, and fallback behavior.
- Check profiling or load-test need.
- Add structured logs with stable correlation fields.
- Add metrics with controlled cardinality.
- Add traces across meaningful boundaries.
- Define alerts, dashboards, backup, recovery, and incident runbook updates.
