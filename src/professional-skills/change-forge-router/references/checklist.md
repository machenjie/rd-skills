# Routing Checklist

- Classify request type using the supported change-type list.
- Mark complexity: L1 isolated local, L2 single-module, L3 multi-module product, L4 product-grade high-risk, or L5 regulated/financial/Web3/AI/migration/production-critical.
- Mark risk: low, medium, high, or critical.
- Identify execution mode, product area, code area, runtime profile awareness, and domain extension signals.
- Identify affected areas: product behavior, UX, domain, API, data, frontend, backend, integration, security, testing, reliability, delivery, documentation.
- Choose the minimum sufficient skill path.
- Select foundation capabilities only when they support a selected professional skill.
- Select domain extensions only when domain signals are present.
- Name quality gates required before handoff and mark skipped gates with a reason.
- Identify dependencies, ordering constraints, rollback needs, release strategy, and evidence requirements.
- Capture blocking questions, non-blocking questions, and assumptions separately.
- Produce the required ChangeForge Routing Result structure.
