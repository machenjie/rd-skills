# Domain Checklist

- Name affected entities, value objects, aggregates, and policies.
- Identify existing and new invariants.
- Map valid and invalid state transitions.
- Identify ownership, tenancy, and permission rules.
- Identify domain events added, removed, reordered, or changed.
- Check audit trail and historical meaning.
- Identify consistency boundaries and transaction needs.
- Define domain-level tests and regression fixtures.
- Document rule authority and consumer impact.
