# Intake Checklist

- Capture request source and stakeholder intent.
- Describe current behavior using observable evidence.
- Describe desired behavior without unnecessary implementation choices.
- List non-goals and explicit out-of-scope areas.
- Record constraints: compatibility, timing, compliance, platform, budget, dependencies.
- Separate assumptions from missing information.
- Identify user value and business reason.
- Name completion signal and required evidence.
- Mark affected surfaces for impact analysis.
