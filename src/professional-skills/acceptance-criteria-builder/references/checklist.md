# Acceptance Checklist

- State actor, precondition, action, and expected result.
- Cover normal successful behavior.
- Cover invalid inputs and error responses.
- Cover boundary values and state transitions.
- Cover permission, role, tenancy, and ownership cases.
- Cover regression cases tied to current behavior.
- Cover compatibility and API consumer expectations.
- Define product-level verification evidence.
- Mark criteria that require manual sign-off or monitoring evidence.
