# Project Initialization Checklist

- Map each top-level folder to an architectural responsibility.
- Separate source, tests, configuration, docs, build output, and deployment assets.
- Define source entry points and package boundaries.
- Define local and CI verification commands.
- Document required environment variables without secret values.
- Keep generated artifacts out of authored source unless explicitly committed by policy.
- Confirm test layout mirrors behavior and integration boundaries.
- Record unresolved setup risks and ownership.
