# Engineering Stage Professionalism Checklist

- Name exactly one current stage and one next stage or closed state.
- Map each selected capability to current stage, product surface, language surface, or explicit risk trigger.
- Record skipped heavy capabilities with reasons; never silently omit release, security, architecture, testing, or documentation gates.
- Launch `senior-programming-judgment-core` for non-trivial engineering behavior or record an explicit trivial/no-semantic/no-engineering skip reason.
- Confirm repository context, graph, memory, and validation freshness before moving from planning to edit or from validation to handoff.
- Require owner/reviewer separation and targeted re-review after repair.
- State validation command, exit code, artifact/report path, proof limits, rollback note, residual risk, and next owner before closure.
