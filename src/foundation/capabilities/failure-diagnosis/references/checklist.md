# Failure Diagnosis Checklist

- Record symptom, impact, scope, and first detection time.
- Build a timeline of deploys, config changes, data changes, traffic, and external events.
- Gather logs, metrics, traces, diffs, test output, inputs, and affected examples.
- Separate trigger from root cause and contributing factors.
- List hypotheses and how each was confirmed or rejected.
- Define immediate mitigation and permanent fix.
- Add regression prevention through tests, monitors, runbooks, or controls.
- Verify the fix against the original failure mode.
