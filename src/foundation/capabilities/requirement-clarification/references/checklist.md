# Requirement Clarification Checklist

- Record the request source and the exact behavior being requested.
- Separate observed facts from interpretations and solution guesses.
- Mark unknowns as blocking when they can change contracts, data, permissions, lifecycle, compliance, cost, rollout, or recovery.
- Mark unknowns as non-blocking only when they can be isolated without changing public behavior.
- List explicit assumptions with source, owner, or decision record.
- List safe assumptions with the reason they are reversible and testable.
- Reject assumptions that create silent data, security, workflow, or compatibility risk.
- State whether coding is blocked, partially allowed, or fully allowed.
- Define the minimum safe implementation scope.
- Name the owner who can answer each blocking question.
