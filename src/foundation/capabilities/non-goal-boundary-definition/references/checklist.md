# Non-Goal Boundary Definition Checklist

- State the approved in-scope behavior.
- State out-of-scope behavior using concrete surfaces and workflows.
- Define current version boundary and deferred version work.
- Identify adjacent redesigns, migrations, and platform changes that are excluded.
- Identify assumptions that must not influence code, schema, API, UI, or permissions.
- Confirm exclusions do not bypass required safety or compatibility work.
- Add review checks that detect unauthorized expansion.
- Add acceptance checks that prove excluded behavior did not change.
- Record owner for deferred decisions.
- Ensure non-goals are visible to planning and review.
