# Architecture Style Selection Checklist

- State the current architecture and the proposed style.
- List the hard constraints and credible near-term constraints.
- Test whether a modular monolith satisfies the need before selecting microservices.
- Justify any distributed boundary with deployment, scaling, ownership, or fault isolation evidence.
- Define data ownership and transaction implications.
- Define deployment, observability, and incident ownership impact.
- Identify rejected alternatives and why they fail.
- Name migration steps and rollback or containment options.
- Record risks, mitigations, and review owner.
