# Relational Database Checklist

- Confirm relational storage is justified by transactions, integrity, consistency, constraints, or complex SQL queries.
- Define source of truth, tables, relationships, and ownership boundaries.
- Define primary keys, foreign keys, uniqueness, check constraints, and nullability.
- Define transaction scope, isolation level, locks, and conflict behavior.
- Match indexes to predicates, joins, sorting, cardinality, and write cost.
- Review migration phases, lock risk, rollback or containment, and validation queries.
- Define query performance evidence and production observability.
- Define tests for constraints, concurrency, migrations, and repository behavior.
