# Context Packaging Checklist

- State task goal, scope, and non-goals.
- Include only relevant requirements, decisions, files, and contracts.
- Separate verified evidence from inference.
- List architecture boundaries and ownership constraints.
- List required tests, quality gates, and acceptance evidence.
- Record open questions separately from accepted facts.
- Exclude secrets, credentials, and unrelated repository history.
- Add freshness markers and drift triggers that require revalidation.
