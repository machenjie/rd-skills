# Prototype Description Checklist

- State the user intent and task supported by the UI.
- Identify actors and role-specific differences.
- Define content hierarchy and decision order.
- Define page, panel, modal, or inline layout regions.
- Define primary, secondary, destructive, and cancel actions.
- Define interaction contract for submit, edit, confirm, retry, and undo when relevant.
- Define validation timing, error placement, and input preservation.
- Define required states without duplicating the full state matrix.
- State existing components and patterns to reuse.
- Flag new component or accessibility decisions requiring review.
- Map changed prototype decisions to test, validator, review artifact, or residual risk.
- State evidence limits for design source, live render, browser/mobile behavior, screen reader, analytics, and user research.
- Name handoff owner for flow, state, design-system, frontend, product, or technical decisions that the brief cannot settle.
