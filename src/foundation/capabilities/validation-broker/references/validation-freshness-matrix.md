# Validation Freshness Matrix

Use this reference when validation selection, result parsing, generated artifact proof, negative evidence, or stop closure needs more detail than the `SKILL.md` body. The body owns routing and gate decisions; this file supplies the concrete matrices and output template.

## Validator Depth

| Depth | Use When | Required Inputs | Evidence Limit |
| --- | --- | --- | --- |
| Narrow | One bounded source, docs, registry, fixture, report, benchmark, generated artifact, or test surface changed. | Exact path, owning surface, targeted command, no broader risk. | Proves only the selected surface and mapped risk. |
| Module | Multiple files inside one subsystem, capability, skill, package, or owner boundary changed. | Owner boundary, affected tests/evals/builds, skipped sibling scope. | Does not prove repository-wide release readiness. |
| Full | Cross-module, registry, hook runtime, package, install, generated artifact, release, migration, security, or adapter surfaces changed. | Full command list, build/install profile, source/generated map, release risk. | Still limited to commands actually run and their freshness. |
| No-validator | Non-material docs/comment-only or read-only answer with no validation claim. | Rationale, changed path, risk ruled out, residual risk. | Cannot support behavior or release correctness claims. |

## Outcome Parsing

| Outcome | Required Evidence | Closure Rule |
| --- | --- | --- |
| `passed` | Exit code/output proves selected scope after final material edit. | May support closure with stated proof limits. |
| `failed` | Non-zero exit, failing assertion, missing expected output, or blocking warning. | Closure is blocked or repair-required. |
| `stale` | Later material edit affects covered scope, input, generated output, or report. | Rerun or disclose not verified. |
| `partial` | Command covers only part of final diff or one risk dimension. | Do not call it full validation. |
| `not-run` | Command was skipped, unavailable, too expensive, or intentionally deferred. | Provide exact command and residual risk. |
| `not-verified` | Evidence channel cannot prove the claim or output is missing/unparseable. | Avoid completion language for that claim. |
| `unsupported` | Adapter/tool cannot surface outcome, timestamp/order, or covered scope. | Name unsupported channel and handoff command. |

## Changed Path To Validation Map

| Changed Item | Typical Validator | Escalation |
| --- | --- | --- |
| Foundation capability `SKILL.md` | `validate-capabilities`, body links, content size, audit, professionalism eval. | Full AGENTS suite when route-level behavior changes. |
| Professional skill `SKILL.md` | `validate-skills`, body links, audit, professionalism eval, routing/agent behavior if route behavior changes. | Full AGENTS suite for professional path changes. |
| Domain extension | `validate-domain-extensions`, registry, audit, professionalism eval. | Full profile build/install validation. |
| Registry or routing rules | `validate-registry`, `eval-routing`, routing coverage. | Full suite when triggers or route_to behavior changes. |
| Hook runtime source | `validate-hooks`, build profiles, runtime reference links, installation validation. | Full suite because runtime support artifacts compile into `dist/`. |
| Generated artifact or report | Source generator command plus report/build validator. | Source-only or generated-only proof is partial. |
| Benchmark/eval fixture | Fixture validator plus matching eval command. | Professional regression and routing coverage when promoted. |
| Package/install output | Build profile plus `validate-installation`. | Release/delivery gate when packaging semantics change. |
| Documentation-only source | Link/content validator or explicit no-validator rationale. | Escalate if docs describe executable setup, API, or release behavior. |

## Freshness Rules

1. A command is current only for covered paths and inputs that did not materially change afterward.
2. Source edits after a pass stale source validators; generated output edits stale build/install validators unless regenerated from source.
3. Registry, routing, hook runtime, package, benchmark, and install-output edits usually require full-suite or profile-aware validation.
4. Reports generated by evals are evidence only when the generating eval command and final source state are known.
5. Lockfile, fixture, schema, config, environment, or test-data edits can stale otherwise green tests.
6. Compaction, memory, or graph summaries can preserve validator facts but cannot prove freshness without command order and changed-path comparison.
7. A later unrelated pass does not erase an earlier failure; the broker must mark the failure repaired, irrelevant with proof, or still blocking.

## Negative Evidence Rules

- Preserve failed commands, stale reports, skipped checks, blocking warnings, missing expected artifacts, unparseable output, unsupported adapters, and contradictory outcomes.
- Classify each item as blocking, repaired, irrelevant-with-proof, deferred-with-owner, or accepted residual risk.
- When a failure is repaired, the repair must be followed by validation that covers the failed scope.
- When a validator is skipped, handoff must state the exact command, reason, owner, and risk of not running it.
- When output is too large to retain, summarize relevant lines and preserve command/outcome/scope without storing full output.

## Stop Closure Status

| Status | Use When | Required Handoff |
| --- | --- | --- |
| `passed` | Current validation covers all material changed paths and risk surfaces. | Evidence limits and residual risk. |
| `failed` | Blocking failure remains. | Repair owner, failing command, next validator. |
| `stale` | Previously passing evidence predates material edits. | Rerun command or not-verified disclosure. |
| `partial` | Some changed paths or risks are covered, others are not. | Covered/uncovered map and owner. |
| `not-run` | Required command was not executed. | Exact command, reason, risk, owner. |
| `not-verified` | Evidence cannot prove the claim. | Claim restriction and verification path. |
| `unsupported` | Runtime/tool lacks evidence channel. | Adapter limitation and alternate command. |
