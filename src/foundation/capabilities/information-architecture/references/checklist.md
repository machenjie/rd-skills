# Information Architecture Checklist

- Identify primary user tasks and decisions.
- Group pages, modules, and content around tasks rather than tables or services.
- Define hierarchy, labels, and navigation placement.
- Identify entry points, cross-links, breadcrumbs, search, and filters.
- Define role-specific visibility and restricted destinations.
- Include empty, archived, unavailable, and permission-denied structural behavior.
- Check that support and audit tasks remain discoverable.
- Identify content that appears in multiple task contexts.
- Confirm labels match user domain language.
- Map structure risks to user-flow or permission follow-up.
