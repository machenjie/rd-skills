# API Contract Design Checklist

- Define operation name, method, path, and resource semantics.
- Define authentication and authorization requirements.
- Define request schema, response schema, and examples.
- Define status codes and stable error codes.
- Define pagination, filtering, sorting, and ordering for collections.
- Define idempotency key scope, retention, and replay behavior for side effects.
- Define compatibility, versioning, and deprecation behavior.
- Confirm the contract can be expressed in OpenAPI or equivalent form.
- Define contract tests for clients and server behavior.
